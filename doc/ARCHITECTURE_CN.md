# PV-LIO-PLUS 系统架构文档

## 1. 系统概述

**PV-LIO-PLUS** 是一个实时激光雷达-惯性里程计（LiDAR-Inertial Odometry, LIO）系统，用于机器人在未知环境中的实时定位与建图（SLAM）。系统基于 [PV-LIO](https://github.com/HViktorTsoi/PV-LIO) 进行开发，融合了 [VoxelMap](https://github.com/hku-mars/VoxelMap)、[VoxelMap++](https://github.com/uestc-icsp/VoxelMapPlus_Public)、[FAST-LIO2](https://github.com/hku-mars/FAST_LIO) 以及 [IKFoM](https://github.com/hku-mars/IKFoM) 等多种先进算法。

### 1.1 核心特性

- **紧耦合LiDAR-IMU融合**：通过迭代扩展卡尔曼滤波器（iESKF）实现LiDAR与IMU的紧耦合融合
- **自适应体素地图**：支持VoxelMap和VoxelMap++两种体素地图算法，通过配置文件灵活切换
- **多激光雷达支持**：兼容 Livox Avia、Velodyne VLP-16、Ouster OS1-64 等多种激光雷达
- **流形状态估计**：状态估计在流形上进行，旋转使用SO(3)表示，重力使用S2流形表示
- **并行计算**：关键算法环节使用OpenMP并行加速

### 1.2 主要改进

相较于原始 PV-LIO，PV-LIO-PLUS 贡献了以下改进：

1. **修复误差传播公式**：修正了PV-LIO中用于计算残差权重的误差传播公式，实现稳定运行
2. **集成VoxelMap++算法**：参考VoxelMap和VoxelMap++的源码与论文，将VoxelMap++集成到PV-LIO框架中
3. **提升稳定性**：解决了VoxelMap++在PV-LIO框架下的稳定性问题

### 2.2 模块职责

| 模块 | 源文件 | 功能描述 |
|------|--------|----------|
| **主控循环** | `src/voxelMapping.cpp` | 整体算法调度、ROS节点管理、数据同步、状态发布 |
| **点云预处理** | `src/preprocess.h`, `src/preprocess.cpp` | 原始点云解析、盲区过滤、特征提取、降采样 |
| **IMU处理** | `src/IMU_Processing.hpp` | IMU初始化、预积分、点云运动去畸变 |
| **状态估计** | `include/IKFoM_toolkit/esekfom/esekfom.hpp` | 迭代扩展卡尔曼滤波器（iESKF）实现 |
| **状态流形** | `include/use-ikfom.hpp` | 系统状态、输入、噪声的流形定义 |
| **VoxelMap** | `include/voxel_map_util.hpp` | 基于八叉树的自适应体素地图，平面拟合与匹配 |
| **VoxelMap++** | `include/voxelmapplus_util.hpp` | 增强体素地图，增量平面更新与并查集融合 |
| **SO(3)数学** | `include/so3_math.h` | 旋转群SO(3)的指数/对数映射，Rodrigues公式 |
| **通用工具** | `include/common_lib.h` | 数据结构定义、数学工具函数、常量定义 |
| **流形工具** | `include/IKFoM_toolkit/mtk/` | 通用流形运算库（MTK），支持SO(3)、S2等 |

### 4.2 线程模型

系统运行在ROS单线程回调模型下，主循环工作流程如下：

```
主线程（main loop）
│
├─ 1. 检查数据缓冲区是否有可用数据
├─ 2. sync_packages() 同步LiDAR和IMU数据
├─ 3. IMU前向传播和点云去畸变
├─ 4. 降采样和协方差计算
├─ 5. 迭代EKF更新（内部使用OpenMP并行）
│     └─ BuildResidualListOMP()：多线程搜索最近平面
├─ 6. 更新体素地图
│     └─ updateVoxelMapOMP()：多线程更新八叉树
├─ 7. 发布里程计、路径、点云
└─ 8. 回到步骤1

回调线程（ROS Spinner）
├─ livox_pcl_cbk()：将LiDAR消息加入缓冲区
├─ standard_pcl_cbk()：将标准点云加入缓冲区
└─ imu_cbk()：将IMU消息加入缓冲区
```

## 6. 坐标系定义

### 6.1 坐标系关系

```
世界坐标系 (World / Map)
│
├── 机体坐标系 (Body / IMU)
│   │
│   └── LiDAR坐标系 (Lidar)
│
│   坐标变换关系：
│   p_world = R_wb · p_body + t_wb
│   p_body  = R_bl · p_lidar + t_bl
│   p_world = R_wb · (R_bl · p_lidar + t_bl) + t_wb
│
│   其中：
│   R_wb, t_wb = 世界到机体的旋转和平移（由EKF估计）
│   R_bl, t_bl = LiDAR到IMU的外参（标定或在线估计）
```

### 6.2 坐标变换函数

| 函数名 | 变换方向 | 说明 |
|--------|----------|------|
| `pointLidarToIMU()` | LiDAR → IMU | 使用外参将点从LiDAR系变换到IMU系 |
| `pointBodyToWorld()` | IMU → World | 使用当前位姿将点从IMU系变换到世界系 |
| `pointLidarToWorld()` | LiDAR → World | 联合外参和位姿将点从LiDAR系变换到世界系 |
| `transformLidar2World()` | LiDAR → World（批量） | 批量变换整个点云 |
| `transformLidarCovToWorld()` | 协方差变换 | 将LiDAR系点协方差传播到世界系 |
